Expand description

Holds the state for a particular connection.

A Protocol creates a session to respond to a particular connection. For example, an IP protocol might create a session to handle messages with a particular pair of local and remote addresses. A session is in charge of appending headers to outgoing messages, deciding which protocol to use for demuxing incoming messages, and keeping track of state such as TCP windows.
